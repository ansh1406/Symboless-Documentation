Getting Started


Symboless as you would've guessed is a complete CLI based application (I don't heve guts to develope a UI. Even if I tried I'd most likely make it worse than CLI version).
So to get started open your cmd/powershell/bash whatever you have.
type the name of application you got after compiling symls.cpp and press enter.
Now you are currently running Symboless and can give any command and get out of it by giving "end" command.
shell > symls ⏎

Syntax


Syntax of Symboless consists of individual expressions (Ofcourse it's just my terminology you call them whatever you want) seperated by line breaks or and keyword. So you can do something like-
syntax.symls print "Hello "
print "World!"
end

or
syntax.symls print "Hello " and print "World!"
end

Both of them will produce same outcome.
And this chaining can be indefinitely long (Well it'll crash if the line got longer than maximum size of c++ string container).
You can end the program with end keyword as stated in the examples.

Variables


Yes!!! We have variables well I can't even imagine a programming language without the feature of variables. And as you've already guessed they can store a single value of certain data type and can change value throughout the program any number of times. In Symboless you can declare variables let keyword followed by data type of variable followed by name of variable.
variable.symls let integer number

You can also initialize the variable with some initial value while declaring it.
In Symboless variable name can only consist 0-9, A-Z and a-z (No underscore) and it can not start with a number.
variable.symls let integer number is 30

Variables can be of any three data type which Symboless support (Details in next section).
You can change thier value by assigining them with a new value like this.
variable.symls let integer number is 30
number is 50

In symboless variables don't have something luxurious like scope or life so to sum it up every variable is global no matter where it's declared in main program, in subroutine, inside a if statements, inside a loop anywhere every variable is global and can be accessed from anywhere in the program throughout the whole execution.

Data Types


Symboless supports three data types:
1- Integer
2- Text
3- Real
Integer can store positive and negative integers and zero.
Text can store any string of characters.
Real can store floating point numbers.
You can initialize and assign them values like this.
data_types.symls let integer number is 30
let text name is "Symboless"
let real pi is 3.14
number is 50
name is "Hello"
pi is 6.28

Arrays


I won't bore with you what arrays are here's the syntax to declare them.
arrays.symls let integer-array numbers-5
let integer-array n2

You can choose to specify their size like first line otherwise Symboless will declare them with a default size which is 64 by default and can be changed using symlsConfig.json file.
You can access elements of an array like this.
arrays.symls let integer-array numbers-5
numbers-3 is 10
print numbers-3

By default every element of- integer array is 0, text array is "" and real array is 0.0 and yes this also can be changed in config file.

Input Output


You can take input using read keyword and print the output using print keyword.
input_output.symls let text name
read name
print name

You can chain both print and read commands using then keyword.
input_output.symls let text name
let integer age
read name then age
print "Name: " then name then " Age: " then age then newline

You can use newline keyword with print to move to next line (Also customizable using configuration file).

Operators


There are two types of operators in Symboless Mathamatical and Logical(not you old AND, OR, NOT but comparison one's yes I'm calling them logical operator cause they are like they are way more fundamental in terms of logic if I say something is equal to something didn't I determine by some logic).
And also every single operator in Symboless can be configured via configuration file.
OPERATOR FUNCTION
plus Add two integeres,reals or strings
minus Subtract two integeres or reals
into Multiply two integeres or reals
upon Divide two integeres or reals
modulo Modulus of two integeres or reals
exponent Calculate exponent of integers and reals
greater-than Greater than comparison
less-than Less than comparison
equal-to Equal comparison
not-equal-to Not equal comparison

Yes we have modulo function for real numbers too!!!
All of these operators can be used like shown below.
operators.symls let integer a is 10
let integer b is 20
let integer c
c is a plus b
c is a minus b
c is a into b
c is a upon b
c is a modulo b
c is a exponent b
if a is greater-than b then skip else stop
if a is less-than b then skip else stop
if a is equal-to b then skip else stop
if a is not-equal-to b then skip else stop

No no no!!! No need to panic I'll tell more about if, then, else, skip and stop later on in branching section just bear with me till then.
There is no operator precedence all operators are equally prioritized and the order of execution is right to left meaning 10 into 20 plus 5 will be equal to 250 and not 205.
Logical operators can only be used for conditional statements inside if or while expressions.
Mathamatical operators can only be used in conditional statements or in assignment expressions (including initialization) you can not do something like.
operators.symls print a plus b
a is b less-than d

The above snippet will throw an error.

Chaining


Since Symboless is an interpreted language it dosen't know what's written after the line it's currently executing so if want it to read the next statement beforehand you can chain multiple statements together.
chaining.symls let integer a is 10 and let integer b is 20 and let integer c is a plus b

You can also give a line break after and keyword Symboless will understand it and treat them as one single statement (and you need not to care about indentation since we are not python, our syntax may be worse than python but we don't declare your code invalid just because of an extra or less space).
chaining.symls let integer a is 10 and
let integer b is 20 and
let integer c is a plus b

Chaining is an essential part of Symboless as it make it possible to have multiple statements inside if and while statements.

Conditional Statements


Symboless have only one conditional statement if then else stop expression. The condition will go between if and then keywords and if the condition is true the expression between then and else will execute and in case of false the expression between else and stop will execute.
conditional_statements.symls let integer a
let integer b
read a then b
if a is greater-than b then print a else print b stop

If you want to execute multiple statements in any case you must chain them with and keyword
You can add line breaks after then and else keywords and Symboless will treat them as a single expression.
conditional_statements.symls let integer a
let integer b
read a then b
if a is greater-than b then
print a else
print b stop

Each if must be conjugated with else you can not have any if alone
If you want to do nothing in case of condition is true or false you can use skip keyword after then or else respectively or you can even leave it empty (well adding an extra skip will make reading the code easier).

Loops


Symboless have only one loop the while do till-here loop and well I think all other types of loop provided in other languages are just extras while is the only thing you ever need (only if you're a good programmer of course).
The condition is provided between while and do keywords and statements are provided between do and till-here keywords.
You can use recheck keyword to recheck the condition at any point and restart the loop from the beginning (yes it's continue I just didn't wanna use same term).
You can use escape keyword to break out of the loop at any point.
loops.symls let integer a is 0
while a is less-than 10 do
print a then newline and
a is a plus 1 till-here

Just like after then and else you can also insert line break after do keyword.
And just like if you can chain multiple statements inside while statement.

Subroutines


Subroutines are reusable blocks of code in Symboless you can call any subroutine using goto keyword.
In symboless subroutines must be declared only after the main code has ended.
a subroutine starts with subroutine keyword followed by it's name and ends with end keyword.
subroutine.symls goto printHello
end
subroutine printHello
print "Hello World!"
end

Since Symboless don't have scopes subroutines can access all variables declared anywhere in the code and main code can also access variables declared inside any subroutine that has already been called.
You can break out of a subroutine by using esacape keyword.
subroutine.symls let integer a is 10
let integer b is 20
goto sum
print "Sum: " then sum
end

subroutine sum
integer sum is a plus b
end

That extra empty line after the ending of main program is just for the asthetics.

Header File


I have also made a c++ class for Symboless which you can access by including the header file provied in includes folder in the installation.
It have Symboless class under symboless namespace
The constructor accepts 3 parameter the Symboless code, a string in which all the inputs which'll be required while execution are provided and a string in which the output will be returned
header_file.cpp #include "symls.hpp"
#include <iostream>
int main(){
std::string code = "let integer a and read a and let integer b and read b and let integer c is a plus b and print c and end";
std::string inputs = "3\n5";
std::string output = "";
symboless::Symboless s(code, inputs, output);
std::cout << output << std::endl;
return 0;
}
